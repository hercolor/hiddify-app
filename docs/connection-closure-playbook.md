# BflyVPN 连接问题收口说明书

**文档状态：** 执行规范（给研发 / 联调 / 测试 / 外部协助者）
**关联：** `docs/product-definition-v0.1.md`
**日期：** 2026-07-16
**原则：** 先归层、再改一层、全量回归；禁止无复现的补丁式修改。

---

## 1. 目的

当前「不能连接」容易变成多模块交叉补丁（会员门禁、订阅同步、配置守护、DNS/路由、平台权限、core 启动）。
本说明书要求把连接问题 **一次性收口** 为：

1. 一条可复述的 **黄金路径**
2. 可区分的 **L0–L6 失败分层**
3. 强制的 **变更纪律** 与 **回归清单**
4. 与产品门禁一致的 **用户文案**（时间 + 设备；不限流量）

---

## 2. 黄金路径（唯一标准成功）

### 2.1 前置条件

| 条件 | 要求 |
|------|------|
| 账号 | 已登录，会话可恢复 |
| 会员 | 未到期；设备数未超限；后端允许连接 |
| 节点 | 本地缓存或同步后至少 1 个可用节点；已选中节点（或默认选中） |
| 平台 | Android：用户可授予 VPN 权限；Windows：若 TUN 则管理员 + Wintun 就绪；iOS：Network Extension 可用 |
| 网络 | 设备能访问面板 API 与节点入口（测试环境需说明） |

### 2.2 成功步骤（三端一致）

1. 冷启动 → 恢复登录态完成（非 `loggedOut` / 非长时间 `initializing`）
2. 进入 **连接** 页，展示当前节点安全名
3. 点击一键连接
4. 状态依次可观测：`preparing` →（如需）`requestingVpnPermission` → `connecting` → **`connected`**
5. 托盘/通知（若有）显示 **BflyVPN**，非上游品牌
6. 点击断开 → `disconnected`，可再次连接

### 2.3 成功判据（产品 + 测试）

| 级别 | 判据 | 首发是否必须 |
|------|------|----------------|
| S1 隧道建立 | UI 状态为已连接；core 报告 Connected | **必须** |
| S2 基础可用 | 通过约定探测方式验证出网及出口路径（由测试环境定义，勿写死敏感域名到日志） | **必须；与 S1 分开报** |
| S3 切换节点 | 已连接时切换节点可重连成功 | 必须（主路径能力） |

**禁止** 仅凭 UI/core 的 Connected 状态判定连接成功；S1 与 S2 必须分别验证并记录。

### 2.4 超时建议

| 阶段 | 建议上限 | 超时表现 |
|------|----------|----------|
| 账号初始化 | 10s | 提示初始化中/失败可重试 |
| 连接前会员刷新 | 8–15s | 可归为网络或登录过期，不可静默 |
| 从点击到 Connected | 60s | 失败态 + 可区分文案 |
| 节点同步（连接前） | 20–30s | 无节点/同步失败文案 |

具体数值可由研发微调，但必须 **有上限、有失败态**，禁止无限转圈。

## 3. 连接管线分层（L0–L6）

任何「连不上」**必须先归主因层**。允许次要层并存，但 PR/修复说明只允许 **主修一层**。

```
用户点击连接
  L0 账号会话
  L1 会员门禁（时间 / 设备 / 后端拒绝）
  L2 节点与订阅同步（无 URL 暴露）
  L3 配置生成与锁定（final config guard）
  L4 系统权限与平台组件
  L5 Core 启动与保活
  L6 已连接后的路由/DNS/出网
```

| 层 | 名称 | 典型现象 | 应查模块（现仓库方向） | 用户文案方向 |
|----|------|----------|------------------------|--------------|
| **L0** | 账号 | 一点连接就「请先登录」；重启后掉登录 | `auth_notifier`、token 存储、启动恢复 | 请先登录 / 登录已过期 |
| **L1** | 门禁 | 有账号但被拦；到期/超限 | `UserSubscription`、`ensureSubscriptionAccessForConnect`、面板 user info | 请开通/续费；设备已达上限 |
| **L2** | 节点 | 暂无节点；切换节点失败 | 节点缓存、`syncNodes`、profile 激活 | 暂无可用节点 / 请稍后重试 |
| **L3** | 配置 | invalid config；智能分流相关挂 | `final_config_guard`、locked core config、profile 解析 | 加速服务启动失败（勿对用户暴露 JSON） |
| **L4** | 权限 | VPN 授权取消；Windows 非管理员；缺 Wintun | VPN permission、`WindowsNetworkModeGuard` | 需要 VPN 权限 / 请以管理员运行 / 重装网络组件 |
| **L5** | Core | 起不来、秒退、服务不可用 | connection repository、平台 service、hcore | 加速服务启动失败 / 节点不稳定 |
| **L6** | 出网 | UI 已连接但打不开网页 | 路由规则、DNS 锁定、分流模式 | 与「连不上」分开：连接正常，访问异常 |

### 3.1 产品门禁与 L1（必须与 v0.1 一致）

| 规则 | 要求 |
|------|------|
| 限制 | **到期时间** + **设备数** |
| 不限制 | **流量** — 不得再以「流量用尽」作为连接拦截主因；后端改完后客户端删除/停用该分支 |
| 设备超限 | **禁止连接** + 明确提示；**不要**默认跳转支付 |
| 未开通/到期 | 禁止连接 + 引导开通/续费（支付闭环） |

**文案映射（产品意图，可微调措辞）：**

| 原因 | 文案 | 主按钮 |
|------|------|--------|
| 未登录 | 请先登录账号 | 去登录 |
| 登录过期 | 登录已过期，请重新登录 | 去登录 |
| 未开通 | 请先开通会员 | 去开通 |
| 已到期 | 会员已到期，请续费 | 去续费 |
| 设备超限 | 登录设备已达上限（当前 x / 上限 y） | 我知道了 / 联系客服 |
| 账号封禁 | 账号不可用，请联系客服 | 联系客服 |
| 无节点 | 暂无可用节点，请稍后重试 | 重试 |
| VPN 权限 | 需要 VPN 权限才能加速 | 去授权 |
| Windows 权限/组件 | 请以管理员运行或重新安装网络组件 | 我知道了 |
| Core/配置 | 加速服务启动失败，请重试 | 重试 |
| 节点不稳 | 当前节点不稳定，请切换节点 | 去节点页 |

现有代码中仍可能存在「套餐流量已用尽」文案（历史逻辑）。**收口时按 v0.1 移除流量拦截**，避免与产品冲突。

## 4. 变更纪律（反补丁）

### 4.1 开工前必须交付（一页纸即可）

修复者在改代码前写出：

1. **复现步骤**（端、账号类型、会员状态、是否首次安装、点击后秒数与现象）
2. **主因层** L0–L6（单选主因）
3. **证据**（脱敏日志 3–10 行关键行；禁止贴完整 auth / subscribe_url / 节点密码 / 完整服务器地址）
4. **计划改动的文件/模块列表** 与 **明确不改的模块**
5. **回归范围**（至少跑第 6 节清单中与本层相关的项）

无 1–3 不得合入连接相关改动。

### 4.2 单次改动约束

| 允许 | 禁止 |
|------|------|
| 一次 PR 主修一层 | 同一 PR 同时改门禁 + DNS + 路由 + UI 大改 |
| 增加可区分错误码/文案 | 用统一「连接失败」吞掉所有异常 |
| 删掉证明有害的旁路 | 再叠一层「特殊情况 if」绕过根因 |
| 补测试或诊断事件（脱敏） | 为过编译关掉校验、`--no-verify` 掩盖 |

### 4.3 与正式产品大改的边界

| 进行中 | 建议 |
|--------|------|
| 连接黄金路径未绿 | 暂停与连接主路径交叉的 UI 大重构、工程改名、路由大搬迁 |
| 可并行 | 产品文档、设计稿、scheme/包名规范、后端支付接口、纯文案品牌（不碰 core 配置） |

## 5. 推荐排查顺序（给执行者）

```
1) 确认能否复现 → 不能复现则收集端/版本/账号状态，不改代码
2) 看点击后 ClientConnectionPhase 停在哪
3) 若未到 preparing：优先 L0/L1
4) 若 preparing 后失败：看是否 access 校验文案 → L1
5) 是否 no nodes / profile null → L2
6) invalid config / guard → L3（导出脱敏配置摘要，不导出密钥）
7) permission / privilege / wintun → L4
8) core start / service → L5
9) Connected 但不通 → L6（单独工单，勿与「点了没反应」混修）
```

### 5.1 日志与诊断要求

- 使用已有诊断缓冲时：**摘要化**节点名、禁止完整 URL
- 每次失败应能对应到：`layer` + `reason_code` + `user_message`（实现上可逐步加，产品要求可追溯）
- Windows / Android / iOS 失败原因 **同一套语义**，允许平台附加说明

## 6. 回归清单（连接收口退出标准）

### 6.1 门禁与账号（L0–L1）

| ID | 用例 | 期望 |
|----|------|------|
| C-01 | 未登录点连接 | 不可连；提示登录；不进 core |
| C-02 | 登录过期 | 不可连；提示重新登录 |
| C-03 | 未开通会员 | 不可连；引导开通；不进 core |
| C-04 | 会员到期 | 不可连；引导续费 |
| C-05 | 设备超限 | 不可连；超限文案；**不**默认进支付 |
| C-06 | 有效会员 | 可通过门禁进入后续流程 |
| C-07 | 流量字段无论多少 | **不得**因流量拦截（与后端新策略一致） |

### 6.2 节点（L2）

| ID | 用例 | 期望 |
|----|------|------|
| C-10 | 登录后自动同步 | 有节点缓存或同步成功；UI 不显示订阅 URL |
| C-11 | 无节点 | 明确无节点；可重试同步 |
| C-12 | 切换节点（未连接） | 仅更新选中 |
| C-13 | 切换节点（已连接） | 重连成功或可区分失败 |

### 6.3 权限与平台（L4）

| ID | 用例 | 期望 |
|----|------|------|
| C-20 | Android 拒绝 VPN 权限 | 可理解提示；可再次发起 |
| C-21 | Android 授予后连接 | 进入 connecting → connected |
| C-22 | Windows TUN 无管理员 | missing privilege 类提示 |
| C-23 | Windows 缺 Wintun | 网络组件提示 |
| C-24 | iOS 扩展异常 | 可理解失败；不崩溃 |

### 6.4 Core 与状态机（L5）

| ID | 用例 | 期望 |
|----|------|------|
| C-30 | 正常连接 | ≤60s 到 connected |
| C-31 | 连接中重复点击 | 忽略或防抖；不卡死 |
| C-32 | 断开 | 到 disconnected；可再连 |
| C-33 | 连接失败后重试 | 不残留半开服务导致二次必败 |
| C-34 | 启动/停止幂等 | 连续操作不崩溃 |

### 6.5 出网（L6，C-40 首发必测）

| ID | 用例 | 期望 |
|----|------|------|
| C-40 | 已连接基础上网 | 按测试环境验证基础出网及出口路径 |
| C-41 | 全局/智能切换（若保留） | 行为符合产品说明；失败单独归 L6 |

### 6.6 三端同日门槛（连接子集）

| 端 | 最低通过 |
|----|----------|
| Android | C-01–C-07、C-10–C-13、C-20–C-21、C-30–C-34、C-40 |
| Windows | C-01–C-07、C-10–C-13、C-22–C-23、C-30–C-34、C-40 |
| iOS | C-01–C-07、C-10–C-13、C-24、C-30–C-34、C-40 |

**退出连接收口：** 三端上述最低集全绿，且无「未归层的已知连不上工单」。

## 8. 与代码现状的收口注意点（供执行者，非授权乱改）

以下为仓库现状观察，用于避免重复踩坑：

| 观察 | 收口含义 |
|------|----------|
| `ConnectionNotifier` 管线长（门禁→同步→起服务） | 失败点必须带层，不能只改最外层 toast |
| 存在流量不可用文案分支 | 与 v0.1 冲突，收口时按产品删除/停用 |
| `final_config_guard` / DNS / 智能路由历史提交多 | L3/L6 高发区；改前要有配置前后对比 |
| `WindowsNetworkModeGuard` | L4 已有能力，优先复用而非新旁路 |
| `ConnectionErrorMapper` | 扩展错误映射时保持用户可读、无技术黑话 |

---

## 9. 完成定义（Definition of Done）

连接收口 **完成** 当且仅当：

1. 黄金路径 S1 与 S2 在 Android / Windows / iOS 可重复成功
2. L0–L6 失败均有稳定用户文案与主按钮策略
3. 流量不再作为拦截条件
4. 设备超限仅提示禁连，不误导支付
5. 第 6 节最低回归集全绿
6. 无进行中的「未归层、纯碰运气」连接补丁
