## 0. 一句话定义

**BflyVPN**（slogan：**蝴蝶 · 自由连接世界**）是面向自有 XBoard 后端的正式商业客户端。用户路径为：登录 → 自动同步节点 → 查看会员 → 选择节点 → 一键加速；会员仅校验**到期时间**与**设备数**；开通/续费经**系统浏览器支付闭环**，经**自定义 scheme** 回 App 并**自动刷新**状态。
**Android + Windows + iOS 在同一天**以完整能力、正式 UI、自有品牌与可维护工程身份上线；**未达 DoD 不上线**。

## 3. 会员与连接门禁

| 维度 | 策略 |
|------|------|
| 时间 | 到期不可连接 → 续费支付 |
| 设备数 | 超限不可连接 → **提示**（首发不做踢设备） |
| 流量 | **不限制**；不作拦截 |
| 连接收口 | 见 `connection-closure-playbook.md` |

| 条件 | 可连 | 动作 |
|------|------|------|
| 未登录 | 否 | 去登录 |
| 未开通 / 到期 | 否 | 去开通/续费 |
| 设备超限 | 否 | 提示超限（不默认支付） |
| 无节点 | 否 | 重试/客服 |
| 均通过 | 是 | 连接流程 |

---

## 4. 支付

| 项 | 内容 |
|----|------|
| 形态 | 系统浏览器 |
| 回跳 | `bflyvpn://` 自定义 scheme |
| 刷新 | 回 App 自动查单 + 刷会员；回前台兜底 |
| 规范 | `package-and-scheme-spec.md` |

## 5. 信息架构

**主 Tab：** 连接 / 节点 / 我的

**禁止用户路径：** 导入订阅、普通技术设置、上游品牌话术

**隐藏：** 诊断（不泄密）

---

## 6. UI 与上线

- 正式应用级 UI，非 Demo
- 上线前补齐最低设计包（Logo、色板、三主页、关键状态）
- 能力 + UI + 去上游 **全部完成** 才上线
- 开源许可证文件合规保留；产品身份自有化

---

## 7. 推荐交付顺序

```
① 连接黄金路径收口
② 门禁对齐（时间+设备，去流量拦截）
③ 支付闭环 + scheme
④ 正式 UI 三端
⑤ 新包名与工程自有化
⑥ 三端同日验收 → 上线
```
